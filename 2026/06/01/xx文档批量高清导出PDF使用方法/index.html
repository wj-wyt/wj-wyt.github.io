本文记录一个用于批量导出xx文档为高清 PDF 的本地脚本方案。

该方案适用于xx文档分享目录中的批量文档导出，特点是直接抓取文档正文的高清 canvas 页面，再合成为 PDF，避免普通截图方式出现空白页、模糊、乱码等问题。

其中：

  • run_xx_doc_export.py：Python 启动器，负责设置 Node 环境、传参、启动导出任务。
  • xx_export_docs.js：核心导出脚本，负责打开xx文档、抓取页面 canvas、生成 PDF。

一般情况下，只需要运行 Python 脚本即可。

八、导出原理

普通网页截图方式在xx文档里容易出问题，因为xx文档正文不是普通 HTML 文本，而是动态渲染出来的 canvas 页面。

本方案的核心逻辑是：

  1. 使用 Playwright 打开xx文档；
  2. 等待文档页面加载；
  3. 滚动 #scrollable 容器，触发所有页面渲染；
  4. 找到正文 canvas：
  1. 将每一页 canvas 转成 PNG；
  2. 使用 pdf-lib 按页合成为 PDF。

为了保证清晰度，浏览器上下文使用：

所以导出的 PDF 比普通截图更清晰。

九、日志和清单文件

输出目录中会生成一些辅助文件：

说明：

  • _doc_list.json：抓到的文档列表。
  • _export_manifest.json：导出记录，包括成功、跳过、失败。
  • _export_stdout.log：标准输出日志。
  • _export_stderr.log：错误日志。

如果某个文档失败，可以优先查看 _export_manifest.json 和 _export_stderr.log。

十、常见问题

1. 导出的 PDF 是空白页

一般是用了普通网页截图方式。

正确方式应该抓：

而不是使用整页截图。

2. PDF 模糊

检查 JS 中的缩放参数：

如果机器性能足够，可以继续提高，但文件会明显变大。

3. 中文文件名乱码

PowerShell 控制台有时会显示乱码，但实际文件名可能是正常的。

如果文件名真的乱码，需要检查脚本文件是否按 UTF-8 保存。

4. Cookie 失效

表现通常是打不开文档、抓不到目录、页面跳登录。

解决方式：

  1. 重新登录xx文档；
  2. 复制新的 Cookie；
  3. 更新 Cookie 文件；
  4. 重新运行脚本。
